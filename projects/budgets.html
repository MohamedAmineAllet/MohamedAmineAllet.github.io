Aperçu

Le projet

BudgETS est une application web fullstack de gestion de finances personnelles développée en équipe de 6 à l'ÉTS. L'application repose sur le concept d'enveloppes budgétaires — allouer son argent dans des catégories de dépenses prédéfinies, avec suivi mensuel.

Le backend est construit avec NestJS et TypeORM, communiquant avec une base de données MySQL hébergée sur Railway. L'authentification est gérée via des tokens JWT sécurisés. Les images de profil sont stockées sur Cloudinary. L'API est documentée avec Swagger.

La conception des entités et la structure de la base de données ont été réalisées avec Visual Paradigm (diagrammes UML et entité-relation), garantissant une architecture de données solide avant l'implémentation.

Authentification

Connexion & Inscription

Le système d'authentification repose sur des tokens JWT. À l'inscription, les informations de l'utilisateur sont envoyées via une requête POST à l'API NestJS, qui les stocke en MySQL après hachage du mot de passe. À la connexion, un token JWT signé est retourné et stocké côté client pour autoriser les requêtes suivantes. Un bouton « Mot de passe oublié » permet de réinitialiser le mot de passe : l'utilisateur entre son courriel, et un email de réinitialisation lui est automatiquement envoyé par le backend NestJS avec un lien sécurisé et temporaire.

Tableau de bord

Page principale

Après connexion, l'utilisateur accède à son tableau de bord personnel. Il visualise son solde du mois, ses différentes enveloppes budgétaires avec un graphique PieChart de répartition, et ses objectifs d'épargne. Toutes les données sont récupérées dynamiquement depuis l'API et persistées en base de données.

Fonctionnalités clés

Enveloppes & Objectifs

Chaque enveloppe et objectif dispose d'une page dédiée avec graphique de visualisation. Les données sont ajoutées dynamiquement via l'API et stockées en base de données MySQL. La page Objectif permet d'inviter d'autres utilisateurs dans un objectif commun — chaque participant peut contribuer à l'épargne, rendant les objectifs partagés plus motivants et atteignables. Un utilisateur peut également se retirer d'un objectif commun à tout moment via le bouton « Quitter ».

Page Enveloppe — liste des dépenses + graphique barres hebdomadaire
Page Objectif — progression épargne + graphique mensuel
Gestion du profil

Page Profil

La page profil affiche les informations personnelles de l'utilisateur récupérées depuis l'API. La photo de profil est stockée sur Cloudinary et référencée en base de données. L'utilisateur peut modifier ses informations ou supprimer son compte, chaque action déclenchant la requête API correspondante.

Architecture

Comment ça fonctionne

⚛
Frontend — React / TypeScript
Interface en React avec TypeScript. Les composants consomment l'API REST via fetch, et l'état est géré localement. Tailwind CSS pour le style.
🔒
Auth — JWT
À la connexion, le backend retourne un JWT signé. Le token est envoyé dans l'en-tête Authorization de chaque requête protégée et vérifié par NestJS.
🗄
Backend — NestJS / TypeORM
API REST construite avec NestJS, TypeORM comme ORM pour communiquer avec MySQL. Base de données hébergée sur Railway. API documentée via Swagger.
🖼
Images — Cloudinary
Les photos de profil sont envoyées à Cloudinary, qui retourne une URL stockée en base de données. Pas de binaires en DB, juste des références URL.
📐
Conception BDD — Visual Paradigm
Diagrammes entité-relation et UML réalisés dans Visual Paradigm avant l'implémentation, définissant les entités, relations et contraintes de la base MySQL.
📡
Déploiement — Railway
La base de données MySQL est hébergée sur Railway. La connexion entre NestJS et Railway se fait via les variables d'environnement TypeORM.
